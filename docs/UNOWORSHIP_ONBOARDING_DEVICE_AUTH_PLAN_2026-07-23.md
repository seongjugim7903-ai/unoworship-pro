# UnoWorship 온보딩·디바이스 인증 흐름 기획

작성일: 2026-07-23
결정 주체: 사용자 (2026-07-23 대화에서 확정)
관련 문서: `UNOWORSHIP_SAAS_ELECTRON_DATA_ARCHITECTURE_PLAN.md`, `UNOWORSHIP_PRO_GITHUB_ELECTRON_RELEASE_RUNBOOK.md`

## 1. 확정된 방향

**"웹에서 등록·구독을 끝내고, 앱에서는 로그인 1회 → 이후 자동" (프로프리젠터 방식).**
정식 첫 다운로드 버전부터 이 구조를 적용한다. 디바이스 인증 정리 작업은 이 문서를 설계서로 사용한다.

```text
[웹: unoworship.kr]
  ① 개인 회원가입 (이메일)
  ② 교회 등록 — 교회 정보 입력, 운영자 승인
  ③ 구독 요금제 결제
  ④ 다운로드 페이지 안내 (구독 활성 교회만)

[앱: UnoWorship Pro 설치 후]
  ⑤ 최초 1회 로그인 → 소속 교회 자동 인식
  ⑥ 장치 이름 입력 → 디바이스 토큰 발급·키체인 저장
  ⑦ 이후 실행은 자동로그인 (로그인 창 없음, 오프라인 30일 유예)
```

핵심 원칙: **교회 정보 입력은 앱이 아니라 웹에서 미리 끝낸다.**
앱을 처음 켜는 순간은 방송실 예배 준비 시간일 가능성이 높다. 방송실에서의
절차는 "로그인 → 장치 이름 → 확인"을 넘지 않게 한다.

## 2. 현재 구현 상태 (2026-07-23 기준)

| 단계 | 상태 | 위치 |
|------|------|------|
| ① 개인 가입 | 뼈대 있음 | `/signup`, `/complete-profile` |
| ② 교회 등록·승인 | 뼈대 있음 | `/signup/church`, `/admin/church-applications` |
| ③ 구독 결제 | 골격만 (결제 연동 없음) | `/billing/checkout`, `/pricing` |
| ④ 다운로드 안내 | 없음 (GitHub Release 수동 안내) | Release: `v0.1.0-win-preview` |
| ⑤⑥ 디바이스 토큰 발급 | 구현됨 | `electron/auth/*`, `/api/auth/device/*` |
| ⑦ 자동로그인 | 구현됨 | safeStorage 키체인, offline 30일 grace, 앱 업데이트에도 유지 |
| 장치 목록·해제 UI | 없음 (서버 revoke 는 있음) | `device_tokens.revoked_at` |

## 3. 계정 보관 안내 (온보딩 화면·문서에 반드시 포함)

교회 현장의 최다 사고는 비밀번호 분실이 아니라 **담당자 교체**다.

1. **교회 공용 이메일로 등록 권장** (예: media@교회도메인)
   — 비밀번호 재설정이 이메일로 오므로, 이메일 접근권이 곧 계정 소유권이다.
   담당자 개인 Gmail 로 등록하면 그 사람이 떠날 때 교회 계정이 잠긴다.
2. **관리자 2인 이상 등록 권장**
   — 역할 구조(admin/crew/member)는 이미 있으므로 교회당 복수 관리자 UI 만 추가.
3. 등록 완료 화면에서 "이 정보를 교회 인수인계 문서에 기록하세요" 안내 표시.

## 4. 장치 관리 (프로프리젠터의 좌석 관리에 해당)

- 웹 워크스페이스에 **등록된 장치 목록** 화면: 장치 이름, 등록일, 마지막 접속.
- **장치 해제** 버튼: 방송실 컴퓨터 교체 시 이전 장치 revoke → 새 장치 등록.
- 서버 측 revoke(`device_tokens.revoked_at`)는 구현되어 있어 UI 만 필요.
- 플랜별 장치 수 제한은 구독 정책 확정 시 함께 결정.

## 5. 디바이스 인증 정리 작업 범위 (다음 작업)

1. 웹: 교회 등록 → 구독 → 다운로드 안내 페이지 흐름 연결
2. 앱: 최초 실행 마법사를 "로그인 + 장치 이름"으로 축소 (교회 정보 입력 제거)
3. 웹: 등록 장치 목록·해제 UI
4. 계정 보관 안내 문구 삽입 (§3)
5. 결제 연동은 별도 트랙 (수동 승인으로 먼저 운영 가능)

선행 조건: 멀티테넌트(church_id) 이관 — 장치·자료가 교회 단위로 격리되어야
이 흐름이 의미가 있다 (`ARCHITECTURE_PLAN` §3, §4 참조).

## 6. 개발 참고

- 개발 머신(맥미니 외)에서는 `UNOLIVE_SKIP_DEVICE_AUTH=1` (dev 전용)로
  인증을 우회하고 창 배치·기능 개발을 진행한다.
- 현재 배포된 프리뷰(v0.1.0-win-preview)는 Supabase 미포함이라 로그인 비활성 —
  게스트(현장 단독) 모드로만 동작하며, 이는 의도된 상태다.
