# UnoLive Field Hardware & Display Readiness - 2026-05-25

이 문서는 `UNOLIVE_FIELD_NETWORK_TROUBLESHOOTING_2026-05-24.md`와 별개로, 교회 현장 시연/설치 때 발생할 수 있는 하드웨어, 스크린, SDI 라인, 카메라, 확장 모니터 변수를 정리하기 위한 문서다.

첫 기준일은 2026-05-30 토요일 시연이다.

## 목적

UnoLive Plus는 저예산 방송실과 중소형 교회를 위해 ATEM/SDI급 스위처 없이도 운영 가능하게 만드는 제품이다. 다만 교회 현장은 스크린 해상도, 프로젝터, 오래된 SDI 라인, HDMI 컨버터, 캡처카드, Mac 확장 모니터 상태가 모두 다르다.

따라서 제품화 방향은 아래 두 가지를 동시에 가져간다.

1. 현장 운영자가 당일 바로 해결할 수 있는 체크리스트와 장비 키트를 준비한다.
2. 반복적으로 발생하는 현장 변수를 UnoLive 설정/진단 기능으로 흡수한다.

## 시연 전 필수 장비 키트

### 기본 키트

- Mac mini 제어 PC
- 송출 노트북 또는 보조 Mac mini
- HDMI 케이블 2m, 5m 각 2개 이상
- USB-C to HDMI 어댑터 2개 이상
- 전원 멀티탭, 접지형 연장선
- 유선 마우스/키보드
- 예비 LAN 케이블
- 휴대용 모니터 또는 작은 HDMI 테스트 모니터

### 현장 변수를 줄이는 장비

- HDMI EDID 에뮬레이터
- HDMI 스케일러 또는 업/다운 컨버터
- HDMI 분배기
- HDMI to SDI 컨버터
- SDI to HDMI 컨버터
- 짧은 테스트용 SDI 케이블
- 예비 BNC 젠더
- USB HDMI 캡처카드 2종 이상
- USB-C 허브는 전원 공급 안정적인 제품으로 준비

### 운영 보조물

- 화면비 테스트 이미지: 16:9, 4:3, 중앙 십자선, 가장자리 안전영역
- 카메라 입력 테스트 섹션
- 검은 배경 + 흰 가사 테스트 섹션
- 대형 글자 테스트 섹션
- 프롬프트 전용 PMT 테스트 섹션
- 현장 IP/장비 역할을 적을 종이 또는 체크시트

## 강대상 스크린 대응

### 가능한 원인

- 프로젝터 또는 스크린이 실제로 16:9가 아니다.
- 프로젝터가 입력 신호를 4:3, zoom, fill, overscan으로 처리한다.
- HDMI/SDI 컨버터가 EDID를 잘못 전달한다.
- macOS가 확장 모니터 해상도를 자동으로 이상하게 잡는다.
- 브라우저 창이 전체화면이 아니거나 확대율이 100%가 아니다.

### 현장 해결 순서

1. 강대상 출력 모니터에서 `1920x1080 60Hz`를 우선 고정한다.
2. 안 되면 `1280x720 60Hz`로 낮춰서 안정성을 확인한다.
3. 브라우저 확대율을 100%로 맞춘다.
4. 전체화면 또는 kiosk 모드로 `/output`을 띄운다.
5. 테스트 패턴으로 상하좌우 잘림을 확인한다.
6. 잘리는 영역은 UnoLive 마스크/안전영역으로 임시 보정한다.
7. 물리 스크린 비율이 다르면 교회별 출력 프로파일로 별도 저장한다.

### 제품화 기능 후보

- 교회별 출력 프로파일 저장
- 출력별 안전영역 설정: 상/하/좌/우 padding, scale, offset
- 화면 보정 마법사: 십자선, 모서리 박스, 글자 크기 테스트
- 프로파일 적용 대상 분리: 강대상 output, 프롬프트 prompt, 브로드캐스트 broadcast
- 시연 모드에서 `현재 출력 해상도`, `브라우저 DPR`, `캔버스 크기` 표시

## 구형 스크린 및 SDI 라인 대응

### 가능한 원인

- HDMI to SDI 컨버터와 SDI to HDMI 컨버터 방향 혼동
- 3G-SDI, HD-SDI, SD-SDI 지원 범위 불일치
- 1080p60 미지원 장비에 1080p60 신호 입력
- RGB/YCbCr 또는 full/limited range 불일치
- 긴 SDI 라인 감쇠 또는 BNC 접촉 불량
- EDID 협상 실패로 Mac이 비정상 해상도 선택

### 현장 해결 순서

1. Mac mini에서 직접 HDMI 모니터 연결해 정상 여부 확인.
2. HDMI to SDI 컨버터 직후 짧은 SDI 케이블과 테스트 모니터로 확인.
3. 기존 교회 SDI 라인을 통과시킨 뒤 다시 확인.
4. 색이 초록/보라로 깨지면 케이블, 컨버터, 컬러 포맷, 해상도를 순서대로 의심한다.
5. 1080p60이 불안정하면 1080p30 또는 720p60으로 낮춰 본다.
6. EDID 에뮬레이터/스케일러를 Mac 출력 앞단에 넣어 신호를 고정한다.

### 제품화 기능 후보

- 출력 진단 페이지에 색상 패턴, 그레이스케일, 테두리 테스트 제공
- 현장별 검증된 해상도/주사율 기록
- 출력 프로파일에 `권장 해상도`, `스케일`, `마스크`, `메모` 저장
- SDI/HDMI 장비 조합별 호환 기록 테이블

## 카메라 영상 대응

### 가능한 원인

- 카메라가 clean HDMI를 내보내지 않음
- 카메라 출력 해상도와 캡처카드 지원 포맷이 다름
- 저가 캡처카드가 특정 브라우저/GPU에서 검은 화면을 냄
- macOS 브라우저 카메라 권한 미허용
- 동일 카메라를 다른 앱이 점유 중
- USB 허브 전원 부족
- 카메라 영상은 정상인데 브로드캐스트 PGM source인 `/output` 창이 열려 있지 않음

### 현장 해결 순서

1. macOS 카메라 권한을 확인한다.
2. 카메라를 다른 앱이 사용 중인지 확인한다.
3. 캡처카드를 허브가 아닌 본체에 직접 연결한다.
4. 카메라 출력을 `1080p30` 기준으로 맞춘다.
5. 브라우저에서 카메라 선택을 다시 한다.
6. UnoLive 카메라 입력 테스트 섹션으로 확인한다.
7. 브로드캐스트 대시보드 PGM은 `/output` 창이 열려 있어야 들어온다는 점을 확인한다.
8. 실패 시 fallback은 카메라 없는 자막/배경 송출 모드로 전환한다.

### 제품화 기능 후보

- 카메라 진단 페이지
- 현재 선택된 카메라 장치명/해상도/프레임 상태 표시
- 캡처카드 연결 끊김 감지 및 사용자 알림
- 카메라 preview fallback: no camera, black, still image
- 인증 캡처카드 추천 리스트

## Mac mini 확장 모니터 대응

### 가능한 원인

- macOS 디스플레이 배열이 현장마다 바뀐다.
- USB-C to HDMI 젠더가 EDID를 다르게 전달한다.
- 전체화면 창이 다른 Space로 이동한다.
- 절전/화면 잠금 후 외부 모니터 순서가 바뀐다.
- output/prompt/control 창 위치가 저장되지 않는다.

### 현장 해결 순서

1. macOS 디스플레이 설정에서 주 모니터와 확장 모니터 배열을 먼저 고정한다.
2. 강대상 output, 프롬프트 prompt, 제어 compose 역할을 종이에 적고 실제 화면에 라벨링한다.
3. 바탕화면 실행 아이콘으로 세 창을 띄운 뒤 위치를 확인한다.
4. 창이 엉뚱한 모니터에 뜨면 모든 창을 닫고 디스플레이 배열을 다시 확인한 뒤 재실행한다.
5. 시연 중에는 절전, 화면보호기, 자동 잠금을 끈다.
6. USB-C 젠더가 불안정하면 다른 포트 또는 예비 젠더로 교체한다.

### 제품화 기능 후보

- 실행 시 현재 연결된 디스플레이 목록 표시
- `output`, `prompt`, `compose` 창 위치 저장
- 디스플레이 변경 감지 후 재배치 안내
- 실행 전 체크 화면: 모니터 3개 감지 여부, 해상도, 역할 매핑
- 시연 모드: 한 번에 output/prompt/compose 창 재정렬

## 당장 가능한 솔루션

1. 교회별 현장 프로파일을 수동 문서로 먼저 남긴다.
2. 강대상 스크린이 비정형이면 마스크/안전영역으로 임시 보정한다.
3. SDI 라인에는 검증된 컨버터와 짧은 테스트 케이블을 반드시 가져간다.
4. 카메라 입력은 1080p30 기준으로 통일해 테스트한다.
5. Mac mini 확장 모니터가 불안정하면 송출 노트북 + 대형 모니터 조합을 우선 fallback으로 둔다.
6. 시연은 반드시 `카메라 포함`, `카메라 제외`, `프롬프트만`, `녹화` 4가지 모드로 사전 테스트한다.

- 현장 체크리스트를 앱 안에서 진행하는 시연 준비 모드

### 제품화

- UnoLive Plus 권장 하드웨어 번들 구성
- 필수/권장/프리미엄 장비 키트 분리
- 시연 전 교회에 보내는 사전 설문지
- 현장 설치 리포트 템플릿
- 실패 유형별 fallback 매뉴얼

## 운영 원칙

- 시연 실패의 상당수는 앱 코드보다 현장 신호 체인에서 발생한다.
- 앱은 현장 변수를 없앨 수 없지만, 진단 순서와 fallback을 줄 수 있어야 한다.
- Plus 버전은 저예산 장비를 존중하되, 검증되지 않은 장비 조합까지 무조건 책임지는 구조로 가면 안 된다.
- 반복되는 현장 이슈는 `네트워크 프로파일`, `출력 프로파일`, `카메라 프로파일`로 제품 안에 흡수한다.
- Pro/Premium에서는 SDI/NDI/ATEM 중심으로 출력 계층을 분리하되, Plus에서 쌓은 현장 프로파일 데이터는 그대로 자산이 된다.
